Serve local LLMs (GGUF, quant, multimodal) on the Roxabi stack. Siblings with `voiceCLI` and `imageCLI`; consumed by **lyra** (LiteLLM library, per-agent `ModelConfig`) and by **Mickael** (claude-code via the existing LiteLLM proxy).

---

## 1. Goal

| | |
|--|--|
| **Problem** | No uniform way to run/serve/swap local LLMs across hosts, route them to lyra agents, and point claude-code at them |
| **Scope v1** | llama.cpp + TurboQuant fork backend; GGUF + multimodal; OpenAI-compatible HTTP on LAN; hot-swap via catalog |
| **Non-goals v1** | vLLM backend, NATS adapter, training, fine-tuning, RAG, embeddings |
| **First test case** | `YTan2000/Qwen3.6-35B-A3B-TQ3_4S` (12.4 GiB, MoE 35B/3B active, TQ3_4S mixed-quant, requires TurboQuant fork) |

## 2. Host topology

| Host | GPU | VRAM | Role | Model budget |
|--|--|--|--|--|
| `roxabitower` (local) | RTX 5070 Ti | 16 GB | dev, on-demand | Qwen3.6-35B-A3B-TQ3_4S (12.4 GiB), 14B Q5, 32B quants |
| `roxabituwer` (prod) | RTX 3080 | 10 GB | always-on | Qwen3-8B-Q4, Qwen3-4B, Gemma-3-4B |

Lyra hub lives on prod → prod catalog is the "always-available" fallback. Local catalog holds the heavy stuff.

## 3. Decisions (settled)

| Choice | Decision | Reason |
|--|--|--|
| Package name | `llmCLI` / `llmcli` | parallel to voiceCLI/imageCLI |
| Model cache | **HF hub** (`~/.cache/huggingface/hub/`) | dedup across projects, `hf download` returns valid path for llama-server, `hf scan-cache` works |
| Backends v1 | **llama.cpp + TurboQuant fork only** | covers TQ3_4S + most GGUF; vLLM adds heavy CUDA dep conflict, deferred as optional extra |
| Engines layer | **Pluggable Protocol** (`start/stop/health/base_url`) | drop-in for vLLM later — no refactor |
| Daemon state | **`dict[str, Instance]` day 1**, CLI defaults to swap | multi-model later is a CLI flag, not a refactor |
| NATS adapter | **Dropped** | everything goes via LiteLLM (lib for lyra, proxy for claude-code) |
| LiteLLM proxy | **Option A — sibling service** (`make litellm` owns it) | proxy already exists at `~/.litellm/`, shared with Fireworks pass-through; llmCLI registers its backends into `~/.litellm/config.yaml` |
| Multi-host | per-host `llmcli.toml` catalog, `--host 0.0.0.0` bind, `--api-key` auth | lyra pins via `ModelConfig.base_url` per agent |
| Supervisor model | single program `llmcli_serve`, catalog-driven hot-swap | parallel to voicecli_tts / imagecli_gen |

## 4. Architecture

### Package layout (parallel to imageCLI)

```
llmCLI/
  src/llmcli/
    cli.py            typer: pull | serve | stop | status | swap | chat | list | register-proxy
    config.py         ~/.config/llmcli/llmcli.toml — catalog + host settings
    daemon.py         AF_UNIX mgmt socket; tracks dict[str, EngineInstance]
    engine.py         Engine Protocol: start(), stop(), health(), base_url
    engines/
      __init__.py
      llamacpp_tq3.py  TurboQuant fork (for TQ3_4S)
      llamacpp.py      vanilla llama.cpp (standard GGUF)
      vllm.py          (phase 2, optional extra: llmcli[vllm])
    litellm_config.py  reads catalog → writes ~/.litellm/config.yaml fragment
  supervisor/
    conf.d/llmcli_serve.conf
    scripts/run_serve.sh
  llmcli.example.toml
  Makefile
  pyproject.toml       uv + hatchling
  README.md
```

### Engine Protocol

```python
class Engine(Protocol):
    def start(self, model_cfg: ModelSpec) -> EngineInstance: ...
    def stop(self, instance: EngineInstance) -> None: ...
    def health(self, instance: EngineInstance) -> bool: ...
    @property
    def base_url(self) -> str: ...  # e.g. http://0.0.0.0:8091/v1
```

### Catalog (`~/.config/llmcli/llmcli.toml`)

Per-host. Local example:

```toml
[host]
bind              = "0.0.0.0"
public_base_url   = "http://roxabitower.lan"
api_key_env       = "LLMCLI_API_KEY"

[models.qwen3_6-35b-a3b-tq3]
engine     = "llamacpp_tq3"
repo       = "YTan2000/Qwen3.6-35B-A3B-TQ3_4S"
file       = "Qwen3.6-35B-A3B-TQ3_4S.gguf"
mmproj     = "mmproj-BF16.gguf"
port       = 8091
vram_gib   = 13
flags      = ["-ngl","99","-c","8192","-ctk","q4_0","-ctv","tq3_0","-fa","on","--jinja"]

[models.qwen3-14b-q5]
engine   = "llamacpp"
repo     = "Qwen/Qwen3-14B-GGUF"
file     = "qwen3-14b-q5_k_m.gguf"
port     = 8092
vram_gib = 11
flags    = ["-ngl","99","-c","8192","-fa","on","--jinja"]
```

Prod catalog pins smaller models (Qwen3-8B-Q4 on :8091, etc.).

### Request flow

```
                                      ┌─────────────────────┐
                                      │ llama-server :8091  │
                                      │ (llmcli_serve)      │  ← managed by supervisor
                                      │  OpenAI API         │     (hot-swap via catalog)
                                      └──────────▲──────────┘
                                                 │
                        OpenAI format            │
                                                 │
  ┌───────────────────────────────┐   ┌──────────┴──────────┐
  │ lyra hub (litellm lib, #665)  │   │ litellm proxy :4000 │
  │  api_base=.../v1              ├──►│  (Anthropic shim)   │◄── claude-code (ccl/ccp)
  │  per-agent ModelConfig        │   │  ~/.litellm/        │    ANTHROPIC_BASE_URL
  └───────────────────────────────┘   └─────────────────────┘
```

## 5. UX / DX

### Mickael — Make targets (Option A, sibling services)

| Target | Action |
|--|--|
| `make llm` | llmcli_serve with default model from catalog |
| `make llm start` / `reload` / `stop` / `status` / `logs` / `errlogs` | supervisor ops |
| `make litellm [action]` | proxy (existing, unchanged) |
| `make llm swap <name>` | hot-swap model via daemon socket |
| `llmcli list` | show catalog + running state + VRAM |
| `llmcli pull <name>` | `hf download` into HF hub cache |
| `llmcli chat <name> "..."` | one-shot OpenAI call, bypasses proxy |
| `llmcli register-proxy` | write/refresh llmCLI block in `~/.litellm/config.yaml`, then `make litellm reload` |
| `llmcli status` | engines running, ports, VRAM, uptime |

### Mickael — claude-code aliases (new profiles)

`~/.claude/settings.json.local`:

```json
{
  "apiKeyHelper": "bash -c 'cat ~/.config/llmcli/api_key'",
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:4000",
    "ANTHROPIC_MODEL": "qwen3.6-35b-a3b-tq3",
    "ANTHROPIC_SMALL_FAST_MODEL": "qwen3.6-35b-a3b-tq3"
  },
  "model": "qwen3.6-35b-a3b-tq3"
}
```

`~/.bash_aliases`:

```bash
ccl()  { _cc_profile local 0 "$@"; }   # local llmCLI (5070 Ti)
ccp()  { _cc_profile prod  0 "$@"; }   # prod llmCLI  (3080)
cccl() { _cc_profile local 1 "$@"; }   # fast
cccp() { _cc_profile prod  1 "$@"; }
```

### Lyra — config-only (no code changes beyond #665 LiteLLMDriver)

```python
ModelConfig(
    backend="litellm",
    model="openai/qwen3.6-35b-a3b-tq3",
    base_url="http://roxabitower.lan:8091/v1",
    api_key="...",  # from llama-server --api-key
)
```

Per-agent routing: agent A → local heavy model, agent B → prod small model. LiteLLM's native fallback list supports graceful degrade when local is off.

## 6. LiteLLM integration (Option A)

llmCLI does **not** own the proxy. It emits/maintains an llmCLI section in `~/.litellm/config.yaml`:

```yaml
model_list:
  - model_name: qwen3.6-35b-a3b-tq3
    litellm_params:
      model: openai/qwen3.6-35b-a3b-tq3
      api_base: http://roxabitower.lan:8091/v1
      api_key: os.environ/LLMCLI_API_KEY

  - model_name: qwen3-8b-q4
    litellm_params:
      model: openai/qwen3-8b-q4
      api_base: http://192.168.1.16:8091/v1
      api_key: os.environ/LLMCLI_API_KEY
```

`llmcli register-proxy` generates this block from all reachable llmCLI catalogs (local + prod) and calls `make litellm reload`.

## 7. Supervisor wiring

Single program, same pattern as voicecli_tts / imagecli_gen:

```ini
# supervisor/conf.d/llmcli_serve.conf
[program:llmcli_serve]
command=%(ENV_HOME)s/projects/llmCLI/supervisor/scripts/run_serve.sh
directory=%(ENV_HOME)s/projects/llmCLI
environment=HOME="%(ENV_HOME)s",PATH="..."
autostart=false                  ; local: on-demand
autorestart=true
startsecs=20
startretries=3
stdout_logfile=%(ENV_HOME)s/.local/state/llmcli/logs/llmcli_serve.log
stderr_logfile=%(ENV_HOME)s/.local/state/llmcli/logs/llmcli_serve_error.log
```

On prod: `autostart=true` (picked up by lyra.service linger).

`make register` from llmCLI uses the hub's `HUB_GEN_MK` + `hub-link-conf` — identical to voiceCLI/imageCLI.

## 8. Costs & overhead

| Component | RAM | VRAM | CPU | Startup |
|--|--|--|--|--|
| llama-server (Qwen3.6-35B-A3B-TQ3_4S, 5070 Ti) | ~1 GB | 13 GB | GPU-bound | ~10 s |
| LiteLLM proxy (existing) | 150–250 MB | — | idle 0% | 3–5 s |
| Proxy per-request latency | — | — | +1–5 ms | — |
| llmcli daemon | ~60 MB | — | idle 0% | ~1 s |

Claimed perf on 5060 Ti (per model card): PP512 ≈ 1832 tok/s, TG128 ≈ 107 tok/s. 5070 Ti should match or exceed.

## 9. Risks & unknowns

| # | Item | Mitigation |
|--|--|--|
| R1 | `turbo-tan/llama.cpp-tq3` fork existence not verified | Verify before scaffolding |
| R2 | Base model `Qwen/Qwen3.6-35B-A3B` may be community-derived — uploader `YTan2000` is not Alibaba | Test quality on standard Q&A before committing |
| R3 | Blackwell (sm_120, 5070 Ti) needs CUDA 12.8+ for build — known friction | Confirm `nvcc --version` on roxabitower early |
| R4 | Prod 3080/10 GB cannot fit 12.4 GiB model — prod catalog must be smaller | Per-host catalog enforces this; `vram_gib` in catalog doubles as budget check |
| R5 | llama-server has no native auth; `--api-key` flag is opaque | Bind `0.0.0.0` + `--api-key` + `api_key` in ModelConfig; LAN trust only, no public exposure |
| R6 | LiteLLM proxy is a shared resource — schema drift could break Fireworks pass-through | `register-proxy` edits a namespaced block; don't touch other entries |

## 11. Open questions

1. **TurboQuant fork status** — verify `github.com/turbo-tan/llama.cpp-tq3` before committing to TQ3_4S path
2. **Default model per host** — Qwen3.6-35B-A3B-TQ3_4S for local is the test case, but is it the long-term default? (Quality on your workloads needs validation)
3. **`make llm` arg UX** — `make llm swap <name>` vs `llmcli swap <name>` (CLI is cleaner; Make is muscle-memory). Recommend: both, Make delegates to CLI
4. **API key lifecycle** — single shared `LLMCLI_API_KEY` across hosts vs per-host vs per-model — recommend single env var for v1, rotate via `~/.config/llmcli/api_key` file
5. **Prod deploy path** — mirror lyra's `make deploy` (git pull → restart on prod)? Yes, same pattern
